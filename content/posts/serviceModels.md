# IaaS
IaaS (Infrastructure as a Service) is a set of compute networking and storage resources that have been virtualized by a vendor so that a user can access and configure them any way they want.
- The cloud provider manages the physical resources, data centres, cooling, power, network and security, as well as computing resources that include servers and storage.
- Customers can create or provision virtual machines in their choice of Region and Zone available from the Cloud provider.
- These VMs typically come pre-installed with the customer's choice of operating system.
- Providers often provide customers the ability to track and monitor the performance and usage of their cloud services.

# PaaS
PaaS (Platform as a Service) takes advantages of all the virtualized resources form IaaS and then just abstracts them away, so the user doesn't have to worry about managing any of those virtualized resources.
- The provider, in addition to the computing resources, also manages the platform infraestructure which includes the operating systems, development tools, databases, and business analytics.
- The user is responsible for only the application code and its maintenance.
- Provide services and APIs that help simplify the job of developers.

# SaaS
Let's jump up to the top with Software as a Service (SaaS). SaaS is just software that you don't have to install in your machine and you don't have to manually update.
- The provider, in addition to the infrastructure and the platform resources, the provider also hosts and manages the applications and data.
- Users simply access the application through a web browser or client.
- No need to install, update, or manage anything locally.
- Ideal for end-users who want fast access to tools without worrying about technical maintenance.
